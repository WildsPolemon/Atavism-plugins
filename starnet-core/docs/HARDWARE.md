# Обладнання каси StarNet Core

Налаштування: **Каса → Меню → Налаштування → Обладнання**

## USB принтер чека (ESC/POS)

- Підключення через USB→COM (Serial) або Web Serial API (Chrome/Edge)
- Швидкість за замовчуванням: **9600 baud**
- Формат: **ESC/POS**, ширина **80 мм**
- Кнопка **Тест друку** у налаштуваннях

## Терези (COM / USB порт)

- Порт: `COM3` (Windows) або `/dev/ttyUSB0` (Linux)
- Параметри: 9600 8N1 (за замовчуванням)
- Протоколи: CAS, Штрих-М, Raw ASCII, Авто
- При додаванні **вагового товару** вага зчитується автоматично (якщо увімкнено)

### Web Serial (браузер)

1. Chrome/Edge на `https` або `localhost`
2. Налаштування → **Підключити COM/USB**
3. Оберіть порт у діалозі браузера

## Термінал Privat24 (ПриватБанк)

Оплата карткою через POS-термінал ПриватБанк.

| Поле | Опис |
|------|------|
| IP терміналу | Напр. `192.168.1.100` |
| Порт | Зазвичай `2000` |
| Merchant ID | ID мерчанта від банку |
| Terminal ID | ID терміналу |

### Режими

- **Міст (рекомендовано)** — локальний сервіс `http://127.0.0.1:8765`
- **TCP** — пряме підключення до терміналу
- **Ручний** — касир підтверджує оплату на фізичному терміналі

На екрані оплати (вкладка **Безготівковий**) з’являється кнопка **Оплата через термінал Privat24**.

## Локальний міст обладнання

Для повної інтеграції з COM/USB на касовому ПК (Windows/Linux):

```bash
cd starnet-core/hardware-bridge
cp config.example.json config.json   # вкажіть COM3, COM4 тощо
npm install
npm start
```

Сервіс слухає `http://127.0.0.1:8765`.

| Ендпоінт | Опис |
|----------|------|
| `GET /scale/weight` | Зчитати вагу з COM/USB терезів |
| `POST /printer/print` | Друк чека ESC/POS на USB-принтер |
| `POST /privat24/pay` | Оплата карткою на терміналі ПриватБанк |
| `POST /privat24/ping` | Перевірка з'єднання з терміналом |

URL мосту змінюється в **Налаштування → Обладнання → URL мосту обладнання**.

Для тесту без фізичного терміналу: `PRIVAT24_DEMO=1 npm start`
