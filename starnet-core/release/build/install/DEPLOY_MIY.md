# ok.txt есть, но 404 в браузере

Файлы **не в корневой папке сайта** (nginx смотрит в другую директорию).

## Быстрое решение

1. panel21 → **mystfall.miy.link** → **Настройки PHP** → скопируйте путь «Корневая директория»
2. Перенесите **все** файлы (ok.txt, install.php, admin/, app/…) **именно в эту папку**
3. Права: папки **755**, файлы **644**
4. Проверка: https://mystfall.miy.link/ok.html

## Частая ошибка

| Где лежит ok.txt | Что сделать |
|------------------|-------------|
| `mystfall.miy.link/public/ok.txt`, а корень без `/public` | Поднять файлы на уровень выше |
| `mystfall.miy.link/ok.txt`, а корень с `/public` | Опустить в папку `public/` |
| `public_html/ok.txt` | Перенести в папку домена `mystfall.miy.link` |

Подробно: **404-FIX.txt** в архиве.
